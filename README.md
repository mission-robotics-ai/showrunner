# Event Host Playbook

Everything we've learned running in-person robotics and AI events, packaged so the next host — us included — starts from a working system instead of a blank page. Built from real events; every run of show, vendor list, and template here has been used at least once with real attendees.

## How to use this repo

Pick your event type under [`event-types/`](event-types/), copy its scaffold into your planning doc, and build from there. The run-of-show files are skeletons on purpose — drop one in, keep the beats, change the specifics. Pull message templates from [`templates/`](templates/) and fill the placeholders. When you need signage, social cards, or a deck, run the system in [`art-dept/`](art-dept/).

## Principles this repo encodes

1. **Run of show is a scaffold, not a script.** The beats and their reasons transfer between events; the clock times don't have to.
2. **Every commitment gets a ledger row.** Sponsor deliverables, prizes, loaner hardware — if it isn't written down with an owner and a deadline, it will be remembered wrong.
3. **Capture starts before the event ends.** Photos, people, and thanks decay on a 48-hour clock. Name the owner of each before doors open.
4. **Sent is not done.** A thank-you, an invoice request, a prize wire — the loop closes on confirmation, not on send.
